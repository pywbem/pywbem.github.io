 24  """ 
 25  Types to represent CIM typed values, and related conversion functions. 
 26   
 27  The following table shows how CIM typed values are represented as Python 
 28  objects: 
 29   
 30  =========================  =========================== 
 31  CIM type                   Python type 
 32  =========================  =========================== 
 33  boolean                    `bool` 
 34  char16                     unicode string or binary bytes, see (1) 
 35  string                     unicode string or binary bytes, see (1) 
 36  string (EmbeddedInstance)  `CIMInstance` 
 37  string (EmbeddedObject)    `CIMInstance` or `CIMClass` 
 38  datetime                   `CIMDateTime` 
 39  reference                  `CIMInstanceName` 
 40  uint8                      `Uint8` 
 41  uint16                     `Uint16` 
 42  uint32                     `Uint32` 
 43  uint64                     `Uint64` 
 44  sint8                      `Sint8` 
 45  sint16                     `Sint16` 
 46  sint32                     `Sint32` 
 47  sint64                     `Sint64` 
 48  real32                     `Real32` 
 49  real64                     `Real64` 
 50  [] (array)                 `list` 
 51  =========================  =========================== 
 52   
 53  (1) CIM string and char16 types are represented as follows: 
 54  In Python 2 as `unicode` (preferred) or `str`; in Python 3 as `str` (preferred) 
 55  or `bytes`. The implementation may decode binary bytes types offered at the 
 56  interface to unicode text types in the internal representation, using "utf-8" 
 57  encoding. 
 58   
 59  Note that constructors of PyWBEM classes that take CIM typed values as input 
 60  may support Python types in addition to those shown above. For example, the 
 61  `CIMProperty` class represents CIM datetime values internally as a 
 62  `CIMDateTime` object, but its constructor accepts `datetime.timedelta`, 
 63  `datetime.datetime`, `str`, and `unicode` (py2 only) objects in addition to 
 64  `CIMDateTime` objects.
